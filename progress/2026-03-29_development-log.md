# Claude Codeと一緒に「プロジェクトダッシュボード」を作った話

## はじめに

複数のシステムが絡み合う大規模開発（Systems of Systems）のプロジェクトマネジメントで、「各システムの進捗を一画面で把握したい」という課題がありました。

そこで、Claude Code（Anthropic公式のAIコーディングCLI）を相棒にして、Webダッシュボードツールを開発しました。この記事では、どんなアプリを作ったのか、AIとどう開発を進めたのかを紹介します。

---

## 作ったもの

**Project Dashboard Tool** — Systems of Systemsの開発状況を一元管理・可視化するWebダッシュボード。

デプロイ先: https://project-dashboard-gamma-ten.vercel.app

### 主な機能

- **サマリービュー**: ステータス集計 + システムカードの一覧表示
- **ネットワーク図**: React Flowでシステム間の依存関係をインタラクティブに可視化
- **一覧テーブル**: 全システムの情報を表形式で一覧表示
- **カンバンビュー**: システムごとにIssue・キーアイテムを看板形式で表示
- **システム管理**: システム/Issue/キーアイテム/依存関係のCRUD
- **履歴・スナップショット**: Weekly単位で状態を記録し、推移をRechartsでグラフ化
- **設定画面**: ステータス・フェーズ・キーアイテムタイプのカスタマイズ
- **データ入出力**: JSONインポート/エクスポート + PNGエクスポート
- **ダークモード**: ライト/ダーク切替対応

### 技術スタック

| カテゴリ | 技術 |
|---------|------|
| フレームワーク | Next.js 15 (App Router, Static Export) |
| UI | React 19 + TypeScript 5 |
| スタイリング | Tailwind CSS v4 + shadcn/ui |
| 状態管理 | Zustand |
| ネットワーク図 | React Flow (@xyflow/react) |
| チャート | Recharts |
| フォーム | React Hook Form + Zod |
| デプロイ | Vercel（Static Export） |

完全クライアントサイドSPAで、サーバーサイド処理なし。データはJSONファイルのインポート/エクスポートで管理するシンプルな設計です。

## 開発の流れ

### Phase 1〜6: 基盤からフル機能まで

最初のフェーズでは、Claude Codeに仕様を伝えて段階的に実装を進めました。

1. **Phase 1**: 型定義（Zodスキーマ）、Zustandストア、データI/O、ウェルカム画面
2. **Phase 2**: ヘッダー/サイドナビ/フッターのレイアウト、サマリー/ネットワーク図/一覧の3タブ
3. **Phase 3**: システム/Issue/キーアイテム/依存関係のCRUD機能
4. **Phase 4**: Weekly履歴スナップショット、Rechartsグラフ2種
5. **Phase 5**: 設定画面（ステータス/フェーズ/キーアイテムタイプ編集）
6. **Phase 6**: JSONエクスポート + PNGエクスポート + ユニットテスト（39件）

ここまでで、ダッシュボードとしての基本機能が一通り揃いました。

---

### カンバンビュー追加

概要ページにはサマリー・ネットワーク図・一覧の3タブがありましたが、「各システムのIssueやキーアイテムを一覧で見たい」という要望から、4つ目のタブとして**カンバンビュー**を追加しました。

**実装内容:**

- 各システムが1つのカラムになり、中にIssue・キーアイテムがステータス別（未対応→対応中→完了）で表示される
- 完了セクションは折りたたみ式
- 横スクロール対応（システム数が多い場合）
- 各カラムヘッダーの「+」ボタンからIssue/マイルストーン/リスク/決定事項/依存関係を直接登録可能
- ジャンル別フィルタリング機能

Claude Codeに「登録されたシステムごとのIssueやキーアイテムなどが看板みたいに表示されるようにしてほしい」と伝えたところ、レイアウトの選択肢（システム別カラム vs ステータス別カラム）をプレビュー付きで提案してくれました。「システム別カラム」を選んで実装してもらい、その後フィルタリングやアイテム追加機能を段階的に追加していきました。

---

### ダークモード対応

ダッシュボードは長時間見るものなので、目に優しいダークモードを実装しました。

**やったこと:**

- `ThemeProvider`をContext + localStorageで自作（`next-themes`は使わず、Static Export対応のシンプル実装）
- ヘッダーに太陽/月アイコンの切替ボタンを配置
- 14ファイルのハードコードされた色（`bg-white`, `text-zinc-500`等）をセマンティック色（`bg-background`, `bg-card`, `text-muted-foreground`等）に置換
- `prefers-color-scheme`でシステム設定にも追従

**Issue優先度カード背景色**も同時に実装。カンバンビューでIssueカードの背景色が優先度で変わるようにしました:
- 高優先度 → 薄い赤背景
- 中優先度 → 薄い黄背景
- 低優先度 → デフォルト

Tailwind CSS v4の`@theme inline`でCSS変数が定義されている環境だったため、`tailwind-merge`がshadcn/uiのCardコンポーネントの`bg-card`と競合する問題がありました。CSS変数（`--priority-high-bg`）とインラインstyleで解決しました。

---

### 週の表示を金曜日基準に

プロジェクト管理ではWeekly単位で進捗を追うため、週の表示フォーマットを改善しました。

**Before**: ISO週番号（`2026-W12`）— 直感的に何日のことかわからない
**After**: 金曜日の日付（`2026/03/27 (金)`）— 週末の振り返りに自然なフォーマット

さらに、`currentWeek`をPC日時から自動取得するように変更。データ読み込み時に今週の金曜日が自動で設定されるようになりました。設定画面には「今週を取得」ボタンも追加。

---

### UI改善3点セット

最後に、使い勝手を高める3つの改善を実施しました。

**1. ヘッダーに今日の日付を表示**

アプリ名の右横に `2026/03/29(土)` のように今日の日付が表示されるようになりました。ダッシュボードを開いた瞬間に「今日は何日か」がわかります。

**2. カンバンのアイテムクリック → モーダル編集**

以前はカンバンのカードをクリックするとシステム詳細のサイドバーが開いてしまい、個別アイテムの編集までの導線が遠い問題がありました。改善後は、カードクリックでそのIssue/キーアイテムの編集ダイアログが直接開くようになりました。

**3. システム一覧からステータス編集**

システム一覧画面のステータス列がドロップダウンに変わり、詳細画面を開かずにその場でステータスを変更できるようになりました。

## Claude Codeとの開発で感じたこと

### 良かった点

- **段階的な実装がスムーズ**: 「Phase 1: 基盤構築」「Phase 2: レイアウト」のように段階を区切って伝えると、一貫した設計で実装してくれる
- **既存コンポーネントの再利用**: 新しい機能を追加する際、既存のダイアログコンポーネントやZustandアクションを自動的に見つけて再利用してくれる
- **選択肢の提示**: カンバンのレイアウトなど、複数のアプローチがある場面でプレビュー付きの選択肢を提案してくれる
- **QAまで一貫**: 実装後にブラウザでの動作確認まで自動で行い、バグがあればその場で修正してくれる

### 工夫が必要だった点

- **dev serverのキャッシュ**: Turbopackのキャッシュが古い状態でQAが走ることがあった。`.next`削除で解決
- **tailwind-mergeの競合**: shadcn/uiのCardコンポーネントとカスタム背景色の競合は、CSS変数 + inline styleで回避
- **Tailwind v4 + ダークモード**: `@theme inline`で定義されたCSS変数がビルド時に静的解決されるか心配したが、実際には`var()`参照がそのまま保持されていた

---

## まとめ

Claude Codeを使うことで、1つのチャットセッションの中で以下を実現できました:

- 基盤構築からフル機能実装まで6フェーズの段階的開発
- カンバンビュー、ダークモード、週表示改善などの機能追加
- UIの使い勝手改善（モーダル編集、インラインステータス変更）
- 39件のユニットテスト
- Vercelへの自動デプロイ

「AIに指示を出して、一緒に作る」という開発スタイルは、特にダッシュボードのような「やりたいことは明確だけど、UIコンポーネントの組み立てが面倒」なプロジェクトと相性が良いと感じました。
