# Project Dashboard Tool - 仕様書

## 1. プロダクト概要

| 項目 | 内容 |
|------|------|
| ツール名 | Project Dashboard Tool |
| 目的 | Systems of Systems の各システム開発状況を一元管理・可視化 |
| ユーザー | プロジェクトマネージャー |
| 更新頻度 | Weekly |
| データ管理 | ローカルJSONファイル（インポート/エクスポート方式） |
| UI言語 | 日本語 |
| デプロイ先 | Vercel（無料枠） |
| 認証 | 不要（データはローカルJSON管理のため機密性を担保） |

---

## 2. 技術スタック

### フレームワーク選定: **Next.js (App Router) + React**

選定理由:

- Vercelとの親和性が最も高く、ゼロコンフィグデプロイが可能
- App Routerによるファイルベースルーティングで画面構成がシンプル
- Static Export（`output: 'export'`）でサーバーレス完全静的サイトとして動作可能 → Vercel無料枠に最適
- SSR不要（データはローカルJSON）なので、SPAとして完結

### 主要ライブラリ

| カテゴリ | ライブラリ | 用途 |
|----------|-----------|------|
| UIフレームワーク | Tailwind CSS v4 + shadcn/ui | モダンで一貫性のあるUI |
| 状態管理 | Zustand | 軽量グローバルストア（JSONデータ保持） |
| ネットワーク図 | React Flow | システム間依存関係の矢印付きネットワーク図 |
| チャート | Recharts | Weekly履歴の推移グラフ |
| PNGエクスポート | html-to-image | サマリービューの画像書き出し |
| アイコン | Lucide React | 一貫したアイコンセット |
| 日付操作 | date-fns | Weekly履歴の日付処理 |
| ドラッグ&ドロップ | @dnd-kit/core | ネットワーク図のノード配置 |
| UUID生成 | crypto.randomUUID() | エンティティのID生成（ブラウザネイティブ） |
| フォーム | React Hook Form + Zod | バリデーション付きフォーム |

## 5. 機能一覧

### 5.1 データ管理

| 機能 | 説明 |
|------|------|
| JSONインポート | ローカルJSONファイルを読み込み、ダッシュボードに反映 |
| JSONエクスポート | 現在のダッシュボードデータをJSONファイルとしてダウンロード |
| PNGエクスポート | サマリービューを画像としてダウンロード |
| 初期状態 | データ未読込時はウェルカム画面 + サンプルデータ読込ボタン |
| バリデーション | インポート時にJSONスキーマを検証、エラー表示 |

### 5.2 システム管理

| 機能 | 説明 |
|------|------|
| システム登録 | 名前、説明、担当者、ステータス、フェーズを入力 |
| システム編集 | 既存システムの全属性を編集 |
| システム削除 | 確認ダイアログ付き削除（関連する依存関係も削除） |
| ステータス変更 | 信号機式セレクタ（🟢🟡🔴⚫）+ コメント入力 |
| フェーズ変更 | プルダウンから選択 |

### 5.3 依存関係管理

| 機能 | 説明 |
|------|------|
| 依存関係追加 | ソース/ターゲットシステム選択、ラベル、タイプ入力 |
| 依存関係編集 | ラベル、タイプ、説明の編集 |
| 依存関係削除 | 確認ダイアログ付き削除 |
| ネットワーク図表示 | React Flowで矢印付きインタラクティブ表示 |
| ノード配置保存 | ドラッグ位置をJSONに保存 |

### 5.4 Issue管理

| 機能 | 説明 |
|------|------|
| Issue追加 | タイトル、ステータス、優先度、担当者、期限、説明 |
| Issue編集 | 全属性の編集 |
| Issue削除 | 確認ダイアログ付き削除 |
| ステータス変更 | Open → In Progress → Closed |
| フィルタ・ソート | ステータス、優先度でのフィルタリング |

### 5.5 開発キーアイテム管理

| 機能 | 説明 |
|------|------|
| キーアイテム追加 | タイプ（マイルストーン/リスク/技術決定/外部依存）選択、タイトル、説明、期限 |
| キーアイテム編集 | 全属性の編集 |
| キーアイテム削除 | 確認ダイアログ付き削除 |
| タイプ別表示 | アイコン付きでタイプ別にグルーピング表示 |

### 5.6 Weekly履歴管理

| 機能 | 説明 |
|------|------|
| スナップショット保存 | 「今週の状況を記録」ボタンで現在の状態をスナップショットとして保存 |
| 推移グラフ | ステータス分布の推移（積み上げエリアチャート） |
| Issue推移 | Open/Closed件数の推移（折れ線グラフ） |
| 週次比較 | 2つの週を選んでdiff表示 |
| サマリー入力 | 週次の全体コメントを記録 |

### 5.7 設定

| 機能 | 説明 |
|------|------|
| ステータスラベル編集 | 信号機ラベル名・色のカスタマイズ |
| フェーズ編集 | フェーズの追加・削除・順序変更 |
| キーアイテムタイプ編集 | タイプの追加・削除・アイコン変更 |
| プロジェクト名変更 | ヘッダーに表示されるプロジェクト名の変更 |

## 6. PNGエクスポート仕様

### 対象: サマリービュー

- html-to-imageライブラリを使用
- エクスポート時にヘッダーにプロジェクト名と日付を付与
- 背景色を白に固定（ダークモードでもPNGは白背景）
- 解像度: 2x（Retina対応）
- ファイル名: `{projectName}_summary_{YYYY-MM-DD}.png`

---

## 7. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | システム数50件程度でもスムーズに動作 |
| レスポンシブ | デスクトップ優先、タブレットでも閲覧可能 |
| ブラウザ対応 | Chrome, Edge, Safari（最新版） |
| アクセシビリティ | キーボードナビゲーション対応、色覚多様性に配慮したステータス表示（アイコン併用） |
| オフライン | 静的サイトのため、一度読み込めばオフラインで動作（Service Worker不要） |

## 9. サンプルデータ
